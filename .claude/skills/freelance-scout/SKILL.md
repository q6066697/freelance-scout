# Freelance Scout

Скилл для работы с инструментом «ИИ-разведчик фриланс-заказов»: сбор вакансий
из сменных источников (hh.ru по умолчанию, «Работа России» и Remotive — опционально),
сохранение в CSV/JSON и анализ через Claude.

## Когда применять

- Пользователь ищет заказы/вакансии или хочет понять спрос на рынке.
- Нужно собрать данные по нескольким поисковым запросам и получить выводы.
- Нужно расширить или настроить сам инструмент.

## Структура проекта

```
main.py              CLI: fetch / list / summary / ask
scout/fetcher.py     запросы к Remotive/hh + нормализация в единую схему
scout/storage.py     сохранение/загрузка CSV и JSON
scout/analyzer.py    сводка и ответы на вопросы через Claude
scout/config.py      поисковые запросы, регион, модель, ключ из .env
data/                собранные данные (gitignored)
```

## Базовый рабочий цикл

1. **Собрать данные:** `python main.py fetch` (источник `-s remotive|hh`, запросы `-q`).
2. **Проверить:** `python main.py list` — таблица собранного.
3. **Проанализировать:** `python main.py summary` — сводка по рынку.
4. **Спросить:** `python main.py ask "вопрос"` — ответ по собранным данным.

Перед `summary` и `ask` убедись, что в `.env` задан `ANTHROPIC_API_KEY`.

## Как расширять

### Поменять поисковые запросы
Правь `DEFAULT_QUERIES_REMOTIVE` / `DEFAULT_QUERIES_HH` в `scout/config.py`
(каждый источник имеет свой набор) или передавай `-q` в `fetch`.
Каждый запрос идёт отдельным обращением к API, результаты дедуплицируются по `id`.

### Выбрать источник
`SOURCE` в `.env` или флаг `-s` (`hh` / `trudvsem` / `remotive`). РФ-источники
(hh, trudvsem) требуют доступа к рунету и могут блокироваться под VPN — это нужно
только для `fetch`. Регион hh — `HH_AREA` (113=Россия).

### Добавить новый источник данных
1. Создай функцию `fetch_query` по аналогии в новом модуле `scout/<source>.py`.
2. Приведи ответ к плоскому словарю с теми же ключами, что в `fetcher._normalize`
   (`id, name, employer, area, salary_from, salary_to, currency, schedule,
   experience, requirement, responsibility, published_at, url`).
3. Зарегистрируй функцию в словаре `SOURCES` в `fetcher.py`.
Единый формат полей обязателен (`id, name, employer, area, salary, job_type,
tags, requirement, published_at, url, source`) — на него опираются `storage` и `analyzer`.

### Изменить логику анализа
Промпты и системная инструкция — в `scout/analyzer.py`. Объём контекста
регулируется параметром `limit` в `_build_context` (по умолчанию 60 вакансий).

## Важные правила

- Источники требуют осмысленного `User-Agent` — он уже задан в `fetcher.HEADERS`.
- Соблюдай вежливые паузы между запросами (уже стоит `time.sleep(0.3)`).
- Никогда не коммить `.env` и собранные данные — они в `.gitignore`.
- Анализ должен опираться только на собранный контекст, а не на общие знания модели.
