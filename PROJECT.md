# Описание проекта: ИИ-разведчик фриланс-заказов

## 1. Задача

Поиск релевантных заказов и вакансий по ИИ-автоматизации, промпт-инжинирингу и
Telegram-ботам отнимает время: нужно вручную листать площадки, отсеивать нерелевантное
и держать в голове, что вообще предлагает рынок. Это повторяющаяся рутина.

**Что автоматизируем:** сбор актуальных вакансий из открытого источника, складирование
их в таблицу и получение по ним выводов и ответов на вопросы на естественном языке —
без ручного листания.

## 2. Целевой пользователь

Фрилансер / начинающий специалист в сфере ИИ и автоматизации, который ищет заказы и
хочет понимать спрос: какие навыки востребованы, какие зарплаты, что брать в работу.

## 3. На входе

- Поисковые запросы — свой набор под каждый источник (Remotive — английские:
  `prompt engineer`, `AI engineer`, `machine learning engineer`, `AI automation`,
  `python developer`, `AI agent`, `chatbot`; hh.ru — русские). Настраиваются.
- Регион поиска (код региона hh.ru, по умолчанию вся Россия).
- Свободный вопрос пользователя для режима Q&A.

## 4. На выходе

- `data/vacancies.csv` и `data/vacancies.json` — собранные и нормализованные вакансии.
- Таблица в терминале со списком найденного.
- Сводка по рынку от Claude: направления, зарплаты, частые навыки, топ подходящих вакансий.
- Ответ на свободный вопрос строго по собранным данным.

## 5. Архитектура и стек

Источник данных сменный, ключ не нужен:
- **hh.ru** (`https://api.hh.ru/vacancies`) — рынок РФ. Источник по умолчанию.
- **«Работа России»** (`https://opendata.trudvsem.ru/api/v1/vacancies`) — гос-портал РФ,
  запасной вариант (часто открыт там, где hh заблокирован).
- **Remotive** (`https://remotive.com/api/remote-jobs`) — международные удалёнки, англоязычные.

РФ-доступ нужен только при сборе (`fetch`); анализ идёт по локальным данным.

LLM-анализ — **Claude через Anthropic API**.

```
main.py (CLI: fetch / list / summary / ask)
        │
        ├── scout/fetcher.py   — запросы к Remotive/hh + нормализация в единую схему
        ├── scout/storage.py   — сохранение/загрузка CSV и JSON
        ├── scout/analyzer.py  — сводка и Q&A через Claude
        └── scout/config.py    — запросы, регион, модель, ключ из .env
```

Стек: Python, `requests`, `anthropic`, `python-dotenv`, `rich`.

Принцип работы анализа близок к RAG: модель отвечает не из общих знаний, а из
локально собранного контекста (реальные вакансии), что повышает точность и актуальность.

## 6. Сценарии использования

1. `python main.py fetch` → собрать свежие вакансии.
2. `python main.py list` → быстро глянуть, что нашлось.
3. `python main.py summary` → получить сводку по рынку.
4. `python main.py ask "Где удалёнка с зарплатой выше 150к и упоминанием n8n?"` → точечный ответ.

## 7. Ограничения MVP

- Три источника (hh.ru, «Работа России», Remotive); другие площадки — в развитии.
- Берётся первая страница выдачи на запрос, без пагинации.
- Анализ ограничен размером контекста (по умолчанию до 60 вакансий в запросе к модели).

## 8. Дальнейшее развитие

- Пагинация и накопление истории вакансий по дням.
- Дополнительные источники (другие API, RSS).
- Фильтры (зарплата, формат работы, ключевые слова) до анализа.
- Обёртка в Telegram-бота с ежедневной рассылкой подходящих заказов.
- Векторный поиск по накопленной базе для полноценного RAG.
